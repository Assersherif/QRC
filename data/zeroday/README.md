# Zero-day evaluation splits (Gap 1)

Leakage-free **held-out-attack** train/test splits of NSL-KDD, in **raw form** — all 41
original features kept untouched. No preprocessing, no PCA: apply your own encoding.
Generated by `build_zeroday_splits.py` (seed 42, reproducible).

## Why this exists

`02_data_encoding.ipynb` used a **random** split, so every attack type appears in both train
and test — it only proves "recognises *known* attacks". Each scenario here removes one attack
type (or whole category) from training and puts it **only in the test set**, so we can measure
detection of an attack **never seen during training**.

## Rules (leakage-free by construction)

- The held-out attack rows appear **only in test**.
- Normal traffic is split 75/25 (train/test) so test has a benign baseline too.
- All other ("seen") attacks go to train.
- Task is **binary `is_attack`** (novelty detection) — a held-out attack can't be a valid
  multi-class label. Verified: no held-out label leaks into train.

## Scenarios

| Folder | Held out of train | Story | Test attack rows |
|---|---|---|---|
| `smurf/` | `smurf` (DoS) | new *variant* of a seen family | 2,646 |
| `probe_all/` | all Probe labels | unseen attack *concept* (port scan) — **headline** | 11,656 |
| `r2l_all/` | all R2L labels | hard app-layer attacks — probes the limits | 995 |
| `neptune/` | `neptune` (SYN flood) | extreme: train keeps almost no DoS | 41,214 |

## Files per scenario

| File | Meaning |
|---|---|
| `train.csv` | training rows: 41 raw features + `label`, `category`, `is_attack` |
| `test.csv` | test rows (held-out attack + 25% normal), same columns |
| `manifest.json` | description, held-out labels, row counts |

`INDEX.json` at this level summarises all scenarios.

The three appended columns: `label` (raw attack name), `category` (5-class family:
normal/DoS/Probe/R2L/U2R), `is_attack` (0/1 target).
